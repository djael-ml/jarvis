Développement de Modules JARVIS

Guide officiel de création et d'intégration d'extensions personnalisées

Introduction

JARVIS utilise un système d'extension dynamique et modulaire. Chaque fonctionnalité (recherche de films, domotique, météo, etc.) est gérée comme un script Python autonome placé dans le dossier modules/.

Le serveur charge ces modules au démarrage et écoute les phrases de l'utilisateur. Lorsqu'un mot-clé correspond à la requête, le module prend la main et génère une réponse personnalisée.

Structure d'un Module

Pour être reconnu par JARVIS, un module doit être un fichier Python (.py) contenant une classe définissant les propriétés suivantes :

  1. name : Le nom public du module (utilisé dans l'interface de gestion).
  2. description : Une brève description de son rôle.
  3. keywords : Une liste de mots-clés en minuscules. Si l'un de ces mots est détecté dans le message de l'utilisateur, le module est déclenché.
  4. execute(text, context) : Une fonction asynchrone qui prend la requête utilisateur et un dictionnaire de contexte, puis retourne une chaîne de caractères (la réponse vocale/écrite).

Exemple de Code : Module Simple Météo

Voici le code source complet d'un module typique à sauvegarder sous modules/meteo.py : 

import random

class MeteoModule:
    # Métadonnées du module
    name = "Météo"
    description = "Fournit les prévisions de température locales fictives"
    keywords = ["météo", "température", "degré", "degrés", "pluie", "soleil"]

    async def execute(self, text, context):
        # Récupérer l'instance WebSocket ou la Base de données si nécessaire
        websocket = context.get("websocket")
        db = context.get("db")
        
        # Logique de traitement simple
        temp = random.randint(15, 30)
        villes = ["Paris", "Lyon", "Marseille", "Bordeaux", "Lille"]
        ville = next((v for v in villes if v.lower() in text.lower()), "votre ville")
        
        # Retourner la réponse textuelle que Jarvis lira à haute voix
        return f"D'après mes capteurs, il fait actuellement {temp} degrés avec un ciel dégagé à {ville}."

L'API de Contexte (context)

Le paramètre context passé à la méthode execute contient des objets clés du serveur Jarvis : 

context = {
    "websocket": websocket_connection, # Connexion client active
    "db": db_manager                   # Gestionnaire SQLite de base de données
}

Vous pouvez utiliser context["websocket"] pour envoyer des messages interactifs spécifiques au client (par exemple pour afficher une image, lancer une action sur la page, ou modifier le statut de la sphère).

Exemple pour afficher une image capturée à l'écran : 

# Envoyer un signal d'affichage d'image au client web
await websocket.send_json({
    "type": "show_image",
    "url": "https://example.com/map.jpg"
})

Comment Installer votre Module ?

Il existe deux manières simples d'intégrer vos nouveaux modules :

  1. Importation via l'UI : Allez dans la configuration de Jarvis (raccourci /settings ou tapez /mod), ouvrez l'onglet Modules, cliquez sur "Choisir un fichier", sélectionnez votre fichier .py et cliquez sur "Importer".
  2. Copie Directe : Copiez directement votre fichier dans le sous-dossier modules/ de votre installation Jarvis, puis cliquez sur "Recharger les modules" ou redémarrez le serveur.
Astuce de Sécurité
Pour des raisons de stabilité, tout plantage dans la méthode execute() d'un module est intercepté automatiquement par le noyau de Jarvis pour éviter l'arrêt du serveur principal.
